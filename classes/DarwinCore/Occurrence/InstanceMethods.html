first_improving_identification() Link

As noted in github.com/gbif/occurrence/issues/89, identifiedBy and identifiedByID are primarily meant to provide attribution to the person who first provided the “correct” identification and not necessarily to track all people who may have agreed with that identification, sp we are only including the person who added the first improving identification that matches the observation taxon

Source: show

263 def first_improving_identification
264   return unless dwc_taxon
265 
266   taxon_id = dwc_taxon.id
267   idents = identifications.select( &:current? ).sort_by( &:id )
268   idents.detect {| i | i.taxon_id == taxon_id && i.category == Identification::IMPROVING }
269 end

    # File lib/darwin_core/occurrence.rb
460 def winning_annotations_for_term( term, options = {} )
461   return [] if annotations.blank?
462 
463   inat_term = options.delete( :inat_term ) || term
464   DarwinCore::Occurrence.annotation_controlled_attributes[term] ||= ControlledTerm.
465     joins( :labels ).
466     where( is_value: false, active: true ).
467     where( "LOWER(controlled_term_labels.label) = ?", inat_term.downcase ).
468     first
469   controlled_attribute = DarwinCore::Occurrence.annotation_controlled_attributes[term]
470   return [] unless controlled_attribute
471 
472   annotations.select {| a | a.controlled_attribute_id == controlled_attribute.id && a.vote_score >= 0 }
473 end
