Class InatContentTypeDetector < Object

  • lib/inat_content_type_detector.rb

Custom Paperclip content detector. Generally we want to rely on the file command to ensure images get the right content type even if the file name is absurd, but it doesn't work for CSS files b/c the file command just views them as text/plain, so here we're making an exception for that, even though this does open us up to some arbitrary text uploads for places where we allow css. It might be necessary to do the same for other other file types in the future.

See config/initializers/paperclip.rb for usage

16 def detect
17   if css?
18     "text/css"
19   else
20     type = type_from_file_command
21     # fix for some .m4a audio files in 3GP format coming from smartphones
22     if type == "application/octet-stream" &&
23        mime_types_for_filename.include?( "audio/mp4" ) &&
24        type_from_mime_magic == "video/3gpp"
25       return "audio/mp4"
26     end
27     # fix for some .m4a audio files that file detects as video
28     if type == "video/mp4" && mime_types_for_filename.include?( "audio/mp4" )
29       return "audio/mp4"
30     end
31     type
32   end
33 end
