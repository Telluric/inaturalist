MetaService is a generalized wrapper for ReST web services that use URLs like this:

http://provider.com/endpoint?method_param=method_name&other_param=other_value

On the verge of copying code for a third of these kinds of wrappers, I figured the pattern was common enough to merit a little bit of consilidation.

MetaService assumes an XML response, and returns Hpricot objects.
